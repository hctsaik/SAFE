0 · 這是什麼？

這是一個與「偵測器」和「類別清單」解耦的驗證層（Safety Net）： 讓一個高靈敏、會誤報的偵測器（YOLO）負責「這裡有沒有東西」， 再用 SAM 去背 + DINOv2 zero-shot 特徵 + 與少量 Golden 標準樣本比對 來回答「這東西是不是我要的目標、是哪一類」。
近（cosine ≥ 閾值）= 真目標放行；遠 = 攔截 False Alarm。免重訓、小資料、類別可增減。
🎯 你不用在這裡訓練任何模型。 採用者帶著自己現有的偵測器（YOLO 或任何 production model）來， SAM 與 DINOv2 都是 zero-shot。你只需提供 3 件「非訓練」輸入：
① Adapter：把你模型的輸出轉成 [x1,y1,x2,y2] 像素框（~10 行） ② Golden：每類丟 10–30 張乾淨樣本圖（不用標座標） ③ 閾值：用一小組標好的框自動校準——或先用預設閾值直接跑。
（build_bank 是「嵌入」不是訓練；calibrate 是「搜尋閾值」不是訓練。）

關鍵心智模型（務必先建立）：

🔍 偵測器 (YOLO)

只負責觸發——框出可疑位置，不分類、不判對錯。 刻意調成高 recall（低 conf），容許多框、誤報。

🧠 驗證層 (SAM+DINO)

負責裁決——去背後抽特徵，與每類 Golden 比 cosine 相似度， 取最像的類別與分數；True/False 與類別都由它決定。

何時該用 / 不該用

✅ 很適合，如果你符合多數：

  • 偵測器容易誤報（高 recall、低 precision），或你想壓低 False Alarm
  • 標註資料少，無法把分類器訓到強健
  • 每個目標類別你能拿出少量（10–30 張）乾淨標準樣本（Golden）
  • 你想免重訓就加一層驗證；或類別會增減（加類別只要加 Golden）
  • 目標物件與「非目標」在外觀上語意可分
✗ 不適合： ① 目標與干擾物本質極相似（例如某個非目標物件的形狀=目標形狀，zero-shot 難分）； ② 需要像素級瑕疵分級（良品 vs 刮痕程度）——那是異常偵測/分割的題目； ③ 完全沒有任何標準樣本可參照。

2 · 適配你的專案：6 步驟

以本 repo（github.com/hctsaik/SAFE）為骨架，照下面改即可。先 git clone 後 pip install -r requirements.txt。

⭐ 你的偵測器就是 YOLO？有更快的零手動策展捷徑 → 直接看 §10 給 YOLO 使用者最方便做法（放權重 + 放標註 → 一鍵）。下面 6 步是通用流程。

1定義類別 + 蒐集 Golden

把你的目標類別的「標準長相」乾淨樣本放進 Workspace/Training/<類別>/，每類 10–30 張即可 （去背或裁切到只剩物件本體最佳）。改 config.yaml 的 classes。

Workspace/Training/
├── widget_a/  img001.png ... (15 張標準樣本)
├── widget_b/  ...
└── widget_c/  ...

2接上你現有的偵測器（不訓練）

你已經有 YOLO 或 production model → 直接用它的輸出，不需在本 repo 訓練。 只要寫一個 Adapter，把你模型的結果轉成像素框清單 [[x1,y1,x2,y2], ...]：

# adapter 範例：把「你的模型」接進驗證層
def my_detector(img):
    preds = your_model.predict(img)          # 你既有的推論
    boxes = []
    for p in preds:
        boxes.append([p.x1, p.y1, p.x2, p.y2])   # 轉成像素 xyxy
    return boxes
# OBB/多邊形 → 取外接矩形；COCO xywh → [x, y, x+w, y+h]；正規化座標 → 乘上影像寬高

建議把偵測器的信心門檻調低一點以提高 recall（寧可多框，誤報交給後級攔）——這只是調參、不是重訓。 沒有偵測器才看若你還沒有偵測器，src/train_yolo.py 可訓一個輕量 YOLO11n 觸發器當起點。

3（可選）接 SAM 去背

已內建：SamSegmenter.segment_crops(img, boxes) 用 bbox 當 prompt 批次去背， mask 過小/失敗自動 fallback 回原始裁切。CPU 用 mobile_sam.pt、GPU 用 sam2_b.pt（自動偵測）。 背景複雜時強烈建議開；背景單純可關以加速。

4建 DINOv2 特徵庫（含增強）

讀 Training/，對每張 Golden 做旋轉（必要）+ 視角增強後用 DINOv2 嵌入，存成 vector_bank.npz。一行指令：

python src/build_bank.py        # 讀 config 的 classes + 增強設定
旋轉增強是關鍵：DINOv2 對旋轉敏感（實測某物件 0°→0.88、90°→0.53）。 增強後各角度穩定 0.85+。見 §5。

5定閾值（校準 或 零校準快速啟動）

路線 A — 零校準快速啟動（沒有標註資料時）：直接用預設閾值（如 0.35）先跑， 看 GUI 的分數直方圖把滑桿拖到「真目標群」與「誤報群」之間，再寫回 config。0 標註即可上手。

路線 B — 自動校準（有少量標註時，更準）：準備一小組標好「真目標 / 非目標」的框， 系統掃 cosine 閾值取最大化 F1 者：

python src/pipeline.py --calibrate
校準集務必涵蓋你會遇到的難度（光影、遮蔽、小物件…）。 只用簡單樣本校準 → 閾值偏高 → 上線後真目標被誤殺。見 §5。

6推論、分發、驗證

python src/pipeline.py --run                      # Check/ → Result/{True,False,No_Detection}
python src/pipeline.py --run --output_mode annotated_full   # 畫框大圖（綠/紅）
python run.py gui                                 # 視覺化檢視 + 即時調閾值

用 §6 的真實指標評估，未達標就回 §1/§4/§5 迭代（整理非目標物件、加增強、調閾值）。

4 · 可重用 API（嵌進你自己的管線）

不想用整個 repo 的目錄流程？三個核心類別可獨立使用：

import sys; sys.path.insert(0, "src")
from common import load_config, SamSegmenter, DinoEmbedder, resolve_env
import numpy as np, cv2

cfg = load_config(); sel = resolve_env(cfg)          # 自動偵測 CUDA/CPU 選模型
sam  = SamSegmenter(sel["sam"], sel["device"], cfg)  # 去背器
dino = DinoEmbedder(sel["dino"], sel["device"])      # DINOv2 嵌入器（回傳 L2-normalized 向量）

# 1) 建你自己的 Golden 向量庫（含旋轉增強）
from build_bank import build; build(cfg)             # -> Workspace/vector_bank.npz

# 2) 對一個偵測框做完整驗證
img   = cv2.imread("frame.jpg")
boxes = my_detector(img)                             # 你的偵測器：回傳 [[x1,y1,x2,y2], ...]
crops = sam.segment_crops(img, boxes)                # 批次去背（一次編碼，CPU 友善）
bank  = np.load("Workspace/vector_bank.npz", allow_pickle=True)
bv, bl, classes = bank["vecs"], bank["labels"], [str(c) for c in bank["classes"]]
thr, topk = cfg["matching"]["threshold"], cfg["matching"]["topk"]

for (crop, sam_ok) in crops:
    v = dino.embed(crop)                             # 768/384 維向量
    sims = bv @ v                                    # cosine（皆已 normalize）
    best, score = -1, -1.0
    for ci in range(len(classes)):
        s = np.sort(sims[bl == ci])[-topk:].mean()   # kNN
        if s > score: score, best = s, ci
    verdict = "TRUE:" + classes[best] if score >= thr else "FALSE(intercepted)"
    print(verdict, round(float(score), 3))

或直接用封裝好的 pipeline.SafetyNet：net = SafetyNet(cfg); recs = net.process(img)。

5 · 調校與避坑（我們實際踩過的坑）

🔄 旋轉敏感 → 旋轉增強

DINOv2 對物件旋轉敏感。對 Golden 做旋轉增強（aug_rotations）讓特徵庫涵蓋各角度。 細長物件（螺絲、膠囊）尤其需要。

🎭 非目標物件 要「語意明確有別」

當作硬負樣本/評測的非目標物件，要挑外觀清楚不同於目標者。 本質相似者（zipper≈screw）zero-shot 難分 → 改擴充該目標的 Golden 或微調 DINO。

🎚️ 閾值被嚴苛樣本拉低

校準集只用簡單樣本→閾值偏高→上線誤殺；某些「不真實的極端劣化」樣本又會把閾值拉太低→放行誤報。 用代表性、混合難度的校準集，且情境要真實（例如反光是局部高光、非整片洗白）。

🧩 去背餵 DINO 的方式

黑底貼上 / 緊裁 / 模糊背景會改變特徵。預設黑底+緊裁；背景與物件同色時可試其他。 中空/強反光物件 SAM 會失效 → 一定要留 fallback（退回原始裁切）。

⚡ CPU 慢 → 預算 / 換規格

ViT-B/14 在 CPU 每次 embed 可能數秒。對策：把與閾值無關的重運算預算成快取 （見 build_gui_cache）、或改 dinov2_vits14、或上 GPU。

📏 評分綁真實指標

別用「角色自評」當分數。用有 GT 的資料量 Precision/Recall/攔截率， 分數才有意義、迭代才有方向。

6 · 驗證方法

把偵測框依與 GT 的關係分類，再算四個指標（定義要對齊，校準與評測共用）：

框的真實身分應該指標
命中真目標 (IoU>0.5)放行 True真目標保留率 = 正確放行 / 全部真目標
非目標（非目標物件/背景）攔截 FalseFalse Alarm 攔截率 = 正確攔截 / 全部誤報
放行的框—Precision = 放行中真的是目標的比例
部分重疊 (0.1~0.5)不計分定位瑕疵，超出驗證層職責
嚴格驗證迴圈（可選但推薦）：定義 N 種嚴苛情境 → 量分 → 未達標就自動調參/補資料/重生情境 → 重測，直到平均達標。本 repo src/evaluate.py 即此機制（達 97.56/100），可直接改情境沿用。

本 repo 實測（工業 AOI）：YOLO 誤報率 ~31% → DINO 攔截率 97.7%、真目標保留率 97.9%、 Precision 69% → 99%，且完全免重訓 YOLO。

7 · 落地情境範例

情境目標類別 (Golden)非目標物件 / 誤報來源備註
工業 AOI 光學檢測各零件/元件型號其他元件、反光、紋理斑本 repo demo；金屬反光要當真實高光處理
零售貨架辨識各 SKU 商品正面其他品牌、價標、空位類別常增減 → 免重訓加 Golden 很適合

8 · 移植檢查清單 + FAQ

  • 確認問題符合 §0 決策清單（高誤報、小資料、有 Golden）
  • 每類放 10–30 張乾淨 Golden 到 Workspace/Training/<類別>/，更新 config.classes
  • 準備/接上高 recall 偵測器（低 conf）
  • 決定是否用 SAM 去背（背景複雜→開）
  • python src/build_bank.py 建特徵庫（開旋轉增強）
  • 備一組代表性、含難度、真實的有標籤校準集
  • python src/pipeline.py --calibrate 定閾值
  • --run 推論分發；用 §6 真實指標評估
  • 未達標：整理非目標物件 / 加增強 / 調校準集 / 換 DINO 規格 → 重測
  • CPU 慢就預算快取或換 ViT-S/14；上線監看誤報率/攔截率

FAQ

Q：一定要用 YOLO 嗎？

不用。任何能輸出 bbox 的偵測器都行（甚至滑動視窗）。驗證層只吃「框」。

Q：要多少 Golden？

每類 10–30 張通常夠（few-shot）。樣本越乾淨、涵蓋越多外觀變化越好；旋轉/光度由增強補。

Q：新增一個類別要重訓嗎？

不用。加一個 Golden 子資料夾、重建特徵庫即可（DINO 是 zero-shot）。這正是本架構的最大優勢。

Q：一定要 GPU 嗎？

不用，CPU 可跑（本 repo 全程 CPU 驗證）。只是慢，用預算快取/ViT-S/14 緩解；量大上 GPU。

Q：DINO 分不出兩個很像的目標怎麼辦？

擴充該類 Golden、提高 topk、或對 DINO 做輕量微調（LoRA/線性探針）。最後手段才動 DINO。

Q：閾值要不要每類不同？

預設全域單一閾值最簡單。若某類特別容易混淆，可擴成每類閾值（進階）。

9 · 整合與部署（接線 / 服務化 / 離線 / 即時）

把驗證層接進你既有系統的常見方式——皆不需訓練：

① 服務化：給其他語言/系統用的 REST 微服務

production 是 C#/Java、或下游只想「給圖→回 True/False」？把驗證層包成一支 FastAPI：

# serve.py  →  uvicorn serve:app --port 8000
from fastapi import FastAPI; import numpy as np, cv2, sys; sys.path.insert(0,"src")
from common import load_config; from pipeline import SafetyNet
app = FastAPI(); net = SafetyNet(load_config())          # 啟動載入一次
@app.post("/verify")                                      # 傳影像 + 你偵測器的框
def verify(image_b64: str, boxes: list):
    img = cv2.imdecode(np.frombuffer(__import__("base64").b64decode(image_b64), np.uint8), 1)
    out = []
    for (crop, ok) in net.sam.segment_crops(img, boxes):
        cls, score = net.match(net.dino.embed(crop))
        out.append({"class": cls, "score": round(float(score),3),
                    "verdict": "TRUE" if score>=net.threshold else "FALSE"})
    return {"results": out}

下游（MES/DB）只要打這支 API、依 verdict 決定收/退，再把結果回寫即可（整合 hook 寫在你那端）。

② 離線 / 氣隙環境（不能下載權重）

在有網路的機器先把權重抓下來、隨專案打包，再複製到離線機：

  • DINOv2：首次 build_bank 會把權重存到 .cache/torchhub/ → 整包複製過去，設 torch.hub.set_dir() 指向它。
  • YOLO/SAM：yolo11n.pt、mobile_sam.pt 等 .pt 隨專案帶。
  • 離線機設環境變數避免連網檢查：HF_HUB_OFFLINE=1。

③ 即時 / 高吞吐

  • 上 GPU：device:auto 會自動用 SAM2 + ViT-B；吞吐大增。
  • 背景單純就關 SAM：直接用偵測框裁切餵 DINO，省掉最貴的去背。
  • 用 dinov2_vits14：embed 快數倍；或對偵測框批次嵌入。
  • 抽格驗證：影片不必逐格驗，觸發到才驗、或每 N 格驗一次。

④ 接線 cookbook（各種偵測器輸出 → 像素 xyxy）

你的輸出格式轉換
YOLO Ultralyticsr.boxes.xyxy.cpu().numpy() 直接用
COCO [x,y,w,h][x, y, x+w, y+h]
正規化 cx,cy,w,h乘影像寬高後轉角點
旋轉框 OBB / 多邊形取外接矩形 cv2.boundingRect(pts)
分割 mask由 mask 取 bbox；或直接當 SAM 的 mask 用

⑤ 想沿用偵測器原本的多類別？（類別融合）

你的偵測器若本身有類別輸出，可當先驗：只比對 DINO 對「該偵測類別」的 Golden， 或把「偵測類別==DINO 類別」當額外通過條件。能進一步壓誤報，但非必要——預設讓 DINO 獨立裁決最單純。

⑥ 極少樣本 / 極小物件

  • 每類 < 5 張 Golden：開好旋轉(+光度)增強補足；閾值取稍寬，靠 GUI 直方圖微調。
  • 物件 < 32px：DINO 會把小圖放大、特徵變弱 → 提高來源解析度、或對該類多收幾張小尺寸 Golden。

⭐ 10 · 給 YOLO 使用者：最方便做法（零手動策展）

如果你已經有一版 YOLO（權重 + 你的標註資料 + data.yaml）， 這是最省事的路：放權重、放標註，一行指令自動生成 Golden—— 不寫 adapter、不手動挑圖、不動原始程式、不重訓。
先釐清「YOLO 的 txt」的角色： 那是訓練用的標註，推論時用不到。但本工具會把它「回收」—— 依 txt 的框把每類物件自動裁出來、用和推論同一個 SAM 去背，當成 Golden 樣本。 一份資料、兩種用途。

三步驟（全程零訓練、不動程式）

1一鍵自動策展

python src/yolo_quickstart.py --data 你的/data.yaml --weights 你的/best.pt
# 沒有 data.yaml 也行：
python src/yolo_quickstart.py --images imgs/ --labels labels/ --names a,b,c --weights best.pt

它自動做三件事：① 把 config.yaml 的 yolo.weights 指向你的 .pt； ② 依你的 txt 把每類物件裁出、去背存成 Golden（預設每類 20 張，可調 --per-class）； ③ 把 classes 設為你 data.yaml 的類別名。

2建特徵庫

python src/build_bank.py        # 純嵌入(非訓練)，含旋轉增強

3跑

cp 你要驗證的影像  Workspace/Check/
python src/pipeline.py --run    # → Result/{True,False,No_Detection}
python run.py gui               # 或開 GUI 視覺化 + 即時調閾值

你給什麼 → 工具做什麼

你提供用途是否必要
YOLO 權重 .pt當偵測器（產生候選框）必要
標註影像 + txt + data.yaml自動裁出每類 Golden + 取類別名建議（最省事）
（或）自己準備好的 Golden 圖放 Workspace/Training/<類別>/ 即可二選一
只想換偵測器、Golden 已自己備好？ 那連工具都不用——只要在 config.yaml 設 yolo.weights: 你的.pt，其餘不動。
重要：本管線把 YOLO 當「觸發器」，會忽略 YOLO 自己的類別—— 類別與 True/False 一律由 DINO 比 Golden 決定。所以你的 YOLO 是單類或多類都沒差， 也不必和你的目標類別一致。
